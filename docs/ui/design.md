# XX Mod Manager - UI 设计文档

> 技术栈：React + TypeScript + Tauri 2
> AI 辅助工具：Agentation + react-scan

## 一、设计原则

1. **内容优先**：模组封面图和预览是核心视觉元素，UI 框架为内容服务
2. **渐进式披露**：首次使用只需配置一次，日常使用一键切换
3. **即时反馈**：所有操作（启用/禁用/切换预设）立即反映到界面状态
4. **美学驱动**：大胆配色 + 流畅动效，让管理 mod 成为愉悦体验

**交互细节：**
- **卡片 hover**：轻微抬起 + 阴影加深，显示快捷操作按钮
- **卡片点击**：打开右侧滑出的详情面板（不离开当前页面）
- **右键菜单**：启用/禁用、编辑信息、加入 Pack、删除
- **拖拽**：将 mod 卡片拖到 Preset 栏 → 添加到该 preset
- **批量操作**：Ctrl+点击多选 → 底部弹出操作栏（启用/禁用/打包/删除）
- **冲突标记**：卡片右上角出现 ⚠️ 图标，hover 显示冲突详情

**筛选维度：**
- 角色（自动识别的角色名）
- 状态（已启用 / 已禁用 / 有冲突）
- 标签（用户自定义 tag）
- 排序（名称 / 添加时间 / 最近使用 / 文件大小）

## 七、状态管理

使用 Zustand 进行全局状态管理：

```
stores/
├── useGameStore       # 当前游戏、游戏配置列表
├── useModStore        # mod 列表、启用状态、筛选条件
├── usePresetStore     # preset/pack 数据
├── useKeyStore        # 快捷键数据
└── useUIStore         # UI 状态（侧边栏展开、详情面板、主题）
```

### 数据流

```
Tauri Backend (Rust / TS Core)
        ↕ invoke / event
Tauri API Bridge (TypeScript)
        ↕
Zustand Stores
        ↕
React Components
```

## 八、技术选型

| 需求 | 选择 | 理由 |
|------|------|------|
| 框架 | React 18+ | AI 工具链兼容、TSX 生态 |
| 桌面框架 | Tauri 2 | Rust 后端、体积小、性能好 |
| 路由 | React Router v6 | 标准选择 |
| 状态管理 | Zustand | 轻量、TypeScript 友好、无模板代码 |
| 样式 | Tailwind CSS + CSS Modules | 原子化样式 + 组件级作用域 |
| 动画 | Framer Motion | 声明式动画、手势支持 |
| 组件库 | 自建 + Radix UI 原语 | 保持美学统一、无头组件提供可访问性 |
| 图标 | Lucide React | 统一风格、tree-shakable |
| 国际化 | i18next | 成熟稳定 |
