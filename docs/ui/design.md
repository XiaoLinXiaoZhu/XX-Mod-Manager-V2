# XX Mod Manager - 最终 UI 设计方案

> 技术栈：React + TypeScript + Tauri 2
> AI 辅助：Agentation + react-scan
> 场景交互细节：见 [scenarios/](./scenarios/) 目录

---

## 一、设计哲学

### 核心意象：全屏卡片桌面

整个应用是一张**全屏、半透明的桌面**。所有的 UI 元素都是桌面上的"物件"——卡片、卡叠、收纳盒、纸条、气泡。没有传统意义上的"页面"或"弹窗"，只有桌面上物件的出现、移动和消失。

**三层实体隐喻：**

| 概念 | 实体隐喻 | 视觉 |
|------|---------|------|
| **Mod** | 一张哑光收藏卡 | 有厚度阴影、封面图、可翻转 |
| **Pack** | 一叠用腰封绑起来的卡片 | 叠放露出边缘、hover 散开、可展开 |
| **Preset** | 一个收纳盒 | 盒盖带标签、可打开/关闭、激活时发光 |

### 五条设计原则

1. **卡片即一切**：每个可交互的东西都是一张卡片或由卡片组成
2. **轻触即改**：编辑 = 翻到背面写字，不跳转、不弹模态框、不需要保存按钮
3. **亮暗即状态**：启用 = 卡片亮起发光，禁用 = 卡片暗淡下沉，一眼看全局
4. **拖拽即组织**：叠卡 = 创建 Pack，装盒 = 创建 Preset，拖拽是主要交互语言
5. **动效即反馈**：每个操作都有物理感的动画回应——落地、翻转、亮灭、滑入

### 三个 Section（顶部滑动条切换）

继承旧版 `SectionSelector` 的 skew 滑块动画，内容通过 `SectionSlider` 水平滑移。

| Section | 内容 | 用户视角 |
|---------|------|---------|
| **仓库** | 游戏选择 + Preset 收纳盒 | "我的桌面上摆了几个收纳盒" |
| **Mod管理** | 筛选面板 + 卡片网格 | "我把所有卡片铺开来挑" |
| **设置** | 路径/存储/外观配置 | "调整桌面环境" |

### 左侧筛选面板

继承旧版 `LeftIndex` 树形目录 + `TagSearch` 标签搜索。

筛选不切换页面——筛选改变时，桌面上不匹配的卡片**沉入桌面**（缩小+淡出+下沉），匹配的卡片**归拢重排**，整个过程是连续的桌面变化。

## 三、卡片系统

### 3.1 Mod 卡片

**正面（日常态）：**

```
┌─────────────────────────┐
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │  ← 封面图
│ ▓▓▓▓▓ 哑光材质 ▓▓▓▓▓▓▓ │
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │
├─────────────────────────┤
│  角色名                  │  ← 信息条
│  Mod 名称          [●]  │  ← 状态灯
└─────────────────────────┘
  ▔▔▔▔▔▔▔▔▔▔▔▔▔▔▔▔▔▔▔▔▔    ← 厚度阴影
```

**四种交互态：**

| 状态 | 视觉效果 |
|------|---------|
| **静止** | 哑光质感，微妙内阴影，柔和投影 |
| **Hover** | 根据鼠标位置 3D 倾斜（perspective），表面光泽扫过，轻微抬起 |
| **拖拽** | 放大 + 深阴影 + 跟随鼠标物理旋转（继承旧版 AnimatedElement） |
| **翻转** | Y轴 3D 旋转到背面，可见卡片侧边厚度（继承旧版 FlippableElement） |

**亮/暗状态：**

| 启用（亮牌） | 禁用（暗牌） |
|-------------|-------------|
| 正常亮度 | 整体降低亮度（暗色叠层） |
| 发光边框（主题色） | 无发光 |
| 浮起（阴影深） | 贴桌面（阴影浅） |
| 状态灯 ● 亮 | 状态灯 ○ 灭 |

**启用/禁用操作**：单击卡片正面 → 亮灭切换（光从中心扩散/收缩的动画）。

**背面（编辑态 = "写卡"）：**

点击/滑动翻转到背面，就地编辑：

```
┌──────────────────────────┐
│ ┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈  │
│  名称 (点击即编辑)       │  ← inline 输入，手写风下划线
│ ┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈  │
│  角色: 艾莲              │  ← 点击弹出选择气泡
│  〔泳装〕〔v1〕〔+〕     │  ← 标签贴纸（啪地贴上/撕下）
│                           │
│  ⌨ F1 切换    [改]       │  ← 快捷键，"按下新键"即完成
│  ⚠ 与 EllenV2 冲突       │  ← 只读冲突提示
│                           │
│  📁打开  🗑删除  ●已启用  │  ← 底部操作
└──────────────────────────┘
```

所有修改即时自动保存，无需确认按钮。翻回正面即完成编辑。

### 3.2 Pack（卡叠）

散卡叠在一起 = Pack。

**折叠态：** 最上面一张显示封面，后面的卡片露出边缘偏移。底部显示 Pack 名称（腰封）。

**Hover：** 后面的卡片微微散开，暗示内容。

**展开态：** 卡片扇形散开或平铺，可单独操作每张卡片。

**创建 Pack：** 拖一张卡片叠到另一张上方停留 0.5s → 自动合叠 → 弹出命名气泡（纸条折成腰封）。

### 3.3 Preset（收纳盒）

卡叠和散卡装进盒子 = Preset。

**关闭态：** 只显示盒盖标签 + 内容摘要 + 激活状态。

**打开态：** 盒盖滑开，内部可见 Pack（卡叠）和散装 Mod。可拖入/拖出。

**激活：** 双击盒子 → 盒子发光 + 桌面上属于该 Preset 的卡片全部亮起，不属于的熄灭。整体是一波灯光切换效果。

**创建 Preset：** 点击"新建" → 桌面下方升起空收纳盒（盒盖打开）→ 拖入卡片和卡叠 → 命名。

## 四、五个核心场景

> 详细交互文档见 `docs/ui/scenarios/` 目录。以下为摘要。

### 4.1 开箱（导入新 Mod）

`拖入文件` → 包裹落地弹跳 → `点击拆箱` → 纸片翻开，露出空白新卡 + 自动分析小纸条 → `写卡`（编辑信息/跳过）→ `收纳`（拖到桌面/卡叠/盒子）。

效率版 1.5s，乐趣版 ~35s。后台同步完成 chunk store 归档。

### 4.2 写卡（编辑 Mod 信息）

翻转卡片到背面 → 在背面就地编辑（名称、角色、标签贴纸、快捷键、备注）→ 翻回正面即完成。

长按正面封面区域 → 毛玻璃叠层换封面。支持多卡批量写卡。

### 4.3 摆桌（浏览与启用/禁用）

桌面铺开所有卡片 → 左侧筛选（不匹配的卡片沉入桌面，匹配的归拢）→ 单击亮灭切换启用/禁用。

冲突时卡片抖动 + 红色连线 + 非阻断小纸条提示。启动游戏时亮牌"集合"飞向启动按钮。

### 4.4 装盒（组织 Pack/Preset）

拖叠两张卡 = 创建 Pack → 拖更多卡到卡叠 = 加入 Pack。新建收纳盒 → 拖入卡片和卡叠 = 创建 Preset → 双击激活 = 桌面灯光切换。

### 4.5 换装（游戏中切换）

全局快捷键唤出"遥控卡"（屏幕角落滑入，always-on-top，半透明）。三个区域：Preset 小卡片切换 / 最近 mod 亮暗灯 / 快捷键备忘。可折叠成一行迷你条。

## 五、浮动层与窗口

### 桌面内浮动层（模拟窗口）

所有编辑和提示都通过浮在桌面上的"物件"完成，不使用传统模态对话框：

| 浮动物件 | 触发 | 形态 |
|---------|------|------|
| 命名气泡 | 创建 Pack/Preset | 小纸条，写完折成腰封/盒盖标签 |
| 冲突纸条 | 启用冲突 mod | 两张卡片间的小纸条，可忽略 |
| 编辑气泡 | 改快捷键/选角色 | 卡片旁边冒出的小气泡 |
| 导入卡片 | 拖入文件 | 包裹 → 拆箱 → 新卡流程 |
| 右键菜单 | 右键卡片 | 浮动操作列表 |

### Tauri 独立窗口

用于需要脱离桌面的场景：

| 窗口 | 触发 | 特征 |
|------|------|------|
| **遥控卡** | 全局快捷键 `Ctrl+Shift+M` | 小窗口，always-on-top，半透明 |
| **Mod 详情窗** | 右键→"在新窗口打开" | 可调大小，支持同时打开多个 |
| **对比窗** | 选两个冲突mod→"对比" | 左右分栏 |
| **快捷键总览窗** | 状态栏/设置打开 | 游戏中置顶查看 |

## 七、状态管理

使用 Zustand，多窗口间通过 Tauri event system 同步。

```
stores/
├── useGameStore        # 当前游戏、游戏列表、配置路径
├── useModStore         # mod 列表、启用/禁用状态、元数据
├── usePresetStore      # preset/pack 层级数据、当前激活 preset
├── useKeyStore         # 快捷键扫描数据、覆写配置
├── useFilterStore      # 筛选状态（树形选中路径、标签、排序）
└── useUIStore          # section 索引、主题、动画强度、窗口状态
```

**数据流：**

```
Tauri Backend (Rust chunk-store / TS Core)
        ↕ invoke / event
Tauri API Bridge
        ↕
Zustand Stores ←──→ Tauri Event (多窗口同步)
        ↕
React Components
```

## 九、视觉规范

### 卡片材质

- **哑光质感**：微妙内阴影（`inset 0 1px 2px rgba(255,255,255,0.05)`）+ 线性渐变模拟表面纹理
- **厚度**：底部加深阴影（`0 4px 0 rgba(0,0,0,0.3)`）模拟卡片厚度
- **三档阴影**：静止（浅）→ hover（中，抬起）→ 拖拽（深，高高抬起）

### 光泽效果

Hover 时一道光泽跟随鼠标扫过卡片表面：`radial-gradient` 跟随鼠标位置的高光叠层。

### 色调

- **深色主题**为默认
- **强调色跟随游戏**：ZZZ = 橙黄、原神 = 青、星铁 = 紫
- 启用态卡片边框使用强调色发光

### 动效预算

| 动画 | 时长 | 缓动 |
|------|------|------|
| 卡片翻转 | 400ms | ease-out |
| 亮灭切换 | 300ms | ease-in-out |
| 筛选重排 | 500ms | spring |
| 拆箱 | 600ms | ease-out |
| 盒子开合 | 400ms | ease-in-out |
| Section 滑移 | 300ms | ease-out |

### 字体

- **标题**：HarmonyOS Sans / 思源黑体，加粗
- **正文**：系统默认 sans-serif
- **卡片背面编辑**：可选手写风格字体增强"写卡"感
