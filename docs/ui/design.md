# XX Mod Manager - 最终 UI 设计方案

> 技术栈：React + TypeScript + Tauri 2
> AI 辅助：Agentation + react-scan
> 场景交互细节：见 [scenarios/](./scenarios/) 目录

---

## 一、设计哲学

### 一句话

**桌面上只有卡片。**

没有页面、没有侧边栏、没有顶部导航条、没有筛选面板。全屏半透明的桌面上，所有东西都是卡片——或者由卡片组成的结构。用户的一切操作就是在桌面上**摆弄卡片**。

### 三层实体隐喻

| 概念 | 实体 | 视觉 |
|------|------|------|
| **Mod** | 哑光收藏卡 | 有厚度、有封面、可翻转 |
| **Pack** | 一叠卡（腰封绑起来） | 叠放、hover散开、可展开 |
| **Preset** | 收纳盒 | 盒盖标签、开合、激活发光 |

### 那些"不是卡片"的东西呢？

**全部变成卡片。**

| 传统 UI | 卡片化 |
|---------|--------|
| 侧边栏筛选面板 | **分类隔板卡** —— 像卡片收藏册里的分类隔板，竖放在桌面上，点击翻开展示类目 |
| 顶部导航/Section切换 | **不需要** —— 桌面就是一张桌子，所有东西同时存在 |
| 设置页面 | **设置卡** —— 一张特殊卡片，翻开是设置面板 |
| 状态栏 | **信息卡** —— 桌面角落的一张小卡片，显示统计 |
| 搜索框 | **搜索卡** —— 桌面上一张空白卡片，写上关键词就开始筛选 |
| 导入弹窗 | **包裹** —— 拖入文件变成一个包裹落到桌面上 |

### 五条原则

1. **卡片即一切**：每个可交互的元素都是一张卡片
2. **轻触即改**：翻到背面写字 = 编辑，不跳转、不弹窗、不需保存
3. **亮暗即状态**：启用 = 亮牌（发光浮起），禁用 = 暗牌（暗淡下沉）
4. **拖拽即组织**：叠卡 = Pack，装盒 = Preset，拖拽是主要交互语言
5. **动效即反馈**：每个操作都有物理感的动画回应

## 二、桌面上的物件

### 2.1 Mod 卡片（核心）

正面：封面图 + 角色名 + mod名 + 亮/暗状态灯。
背面：就地编辑（名称、角色、标签贴纸、快捷键、备注）。

四种交互态：静止 / Hover(3D倾斜+光泽扫过) / 拖拽(放大+深阴影+物理旋转) / 翻转(Y轴旋转见厚度)。

单击正面 = 亮暗切换（启用/禁用）。
翻转到背面 = 写卡（编辑）。

### 2.2 Pack 卡叠

一叠卡片。折叠态只看到最上面一张 + 后面露出的边缘 + 腰封标签。

Hover 后面的卡散开暗示内容。点击展开：扇形散开，每张可独立操作。

创建：拖一张卡叠到另一张上 → 合叠 → 命名腰封。

### 2.3 Preset 收纳盒

盒子。关闭态是一个带标签的盒子。打开：盒盖滑开，内部可见卡叠和散卡。

激活：双击或点击激活 → 盒子发光 → 桌面上所有属于该 Preset 的卡片亮起，其他熄灭。

### 2.4 分类隔板卡

**替代传统侧边栏筛选面板。**

桌面左侧（或用户自定义位置）放着几张竖立的**隔板卡**，就像卡片收藏册/图书馆书架上的分类隔板：

```
  ┌──┐ ┌──┐ ┌──┐ ┌──┐
  │全│ │角│ │类│ │标│
  │部│ │色│ │型│ │签│
  │  │ │  │ │  │ │  │   ← 竖放的隔板卡，像书签/标签页
  └──┘ └──┘ └──┘ └──┘
```

**交互：**
- 点击"角色"隔板 → 隔板向前翻倒/展开，变成一排角色名小卡片标签：

```
  点击前:                     点击后:
  
  ┌──┐                       ┌────────────────────────────┐
  │角│                       │ 艾莲  简  妮可  格莉丝  ... │  ← 隔板展开
  │色│         ──→           │ (点击角色名 = 筛选)         │     成标签条
  │  │                       └────────────────────────────┘
  └──┘
```

- 点击某个角色名 → 桌面上非该角色的卡片沉下去，该角色的卡片归拢
- 再次点击隔板 → 收起，恢复竖立状态，桌面恢复全部显示
- 多个隔板可以同时展开 → 交集筛选

**为什么这比侧边栏好？**
- 它也是桌面上的物件，不是固定的UI框架
- 用户可以拖动隔板到桌面任何位置
- 不需要时可以"推到桌面边缘"甚至"收进抽屉"
- 视觉上与卡片一致，不打破拟物隐喻

### 2.5 搜索卡

桌面上有一张**空白卡片**，它是搜索入口：

```
  ┌─────────────────────────┐
  │                          │
  │   🔍 输入关键词...       │  ← 点击后可输入
  │                          │
  └─────────────────────────┘
```

输入关键词后，不匹配的卡片沉入桌面，匹配的浮起归拢。

清空搜索 → 所有卡片浮回。

搜索卡始终在桌面上可见（通常在左上角或顶部），但它也可以被拖走。

### 2.6 设置卡

一张特殊的卡片——背面是齿轮图标，正面是一张**大号卡片**展开后显示设置选项：

```
  折叠态（在桌面角落）:
  
  ┌────────┐
  │  ⚙️    │
  │  设置   │
  └────────┘
  
  点击展开:
  
  ┌──────────────────────────────────────┐
  │  ⚙️ 设置                    [收起 ▲] │
  │                                       │
  │  游戏: [ZZZ ▾]                        │
  │  Mods目录: D:\...\Mods   [浏览]      │
  │  存储目录: E:\...\Storage [浏览]      │
  │                                       │
  │  存储: 3.8GB  去重率: 86.8%           │
  │  ████████████░░░░  38%                │
  │                                       │
  │  主题: [深色 ▾]   强调色: [跟随游戏]  │
  │  语言: [中文 ▾]   动画: [━━━●━]      │
  │                                       │
  │  版本 0.1.0   [检查更新]              │
  └──────────────────────────────────────┘
```

展开的设置卡就是一张大卡片，浮在桌面上方。背后的卡片仍然隐约可见。

### 2.7 信息卡（替代状态栏）

桌面右下角一张小卡片，始终显示关键信息：

```
  ┌───────────────────────┐
  │ ● 12/56 已启用         │
  │ 💾 3.8GB (去重86%)    │
  │ ⚠ 2 个冲突            │  ← 点击跳到冲突卡片
  └───────────────────────┘
```

这张卡片也可以翻转——背面显示更详细的统计。

### 2.8 游戏卡

桌面顶部有一张（或几张）**游戏卡**——代表当前管理哪个游戏：

```
  ┌──────────────────┐
  │  🎮 绝区零 (ZZZ) │  ← 当前游戏，主题色边框
  └──────────────────┘
```

点击 → 展开一排游戏卡供切换（像 DragableCardList 那样水平排列）。切换游戏 → 整个桌面内容变化（旧卡片滑走，新卡片滑入）。

### 2.9 启动卡

桌面上一张特殊的发光卡片——"启动游戏"：

```
  ╔══════════════════╗
  ║  ▶ 启动游戏      ║  ← 主题色发光
  ║  12 mod 就绪     ║
  ╚══════════════════╝
```

点击后所有亮牌做"集合"飞向启动卡的动画，然后游戏启动。

## 四、五个核心场景（摘要）

> 详细交互文档见 `docs/ui/scenarios/` 目录。

### 4.1 开箱（导入新 Mod）
拖入文件 → 包裹落地弹跳 → 拆箱（纸片翻开） → 空白新卡 + 分析小纸条 → 写卡 → 收纳到桌面/卡叠/盒子。效率版 1.5s。

### 4.2 写卡（编辑 Mod）
翻转卡片到背面 → 在卡片背面就地编辑（名称、标签贴纸、快捷键、备注）→ 翻回即完成，自动保存。

### 4.3 摆桌（浏览与启用/禁用）
桌面铺开卡片 → 分类隔板筛选 → 单击亮灭切换。冲突 = 抖动 + 红线 + 小纸条。

### 4.4 装盒（组织 Pack/Preset）
拖叠两张卡 = Pack → 收纳盒装入卡叠和散卡 = Preset → 双击激活 = 灯光切换。

### 4.5 换装（游戏中切换）
全局快捷键唤出遥控卡（always-on-top 半透明小窗）→ Preset 切换 / Mod 开关 / 快捷键备忘。

## 五、Tauri 独立窗口

桌面内的一切通过卡片和浮动物件完成。但有几个场景需要脱离桌面：

| 窗口 | 触发 | 形态 |
|------|------|------|
| **遥控卡** | 全局快捷键 `Ctrl+Shift+M` | 小窗，always-on-top，半透明 |
| **Mod 详情窗** | 右键→"在新窗口打开" | 独立窗口，可同时开多个 |
| **对比窗** | 选两个冲突mod→"对比" | 左右分栏窗口 |
| **快捷键总览窗** | 信息卡/设置卡打开 | 置顶窗口，游戏中查看 |

## 七、状态管理

Zustand stores，多窗口通过 Tauri event system 同步：

```
stores/
├── useGameStore        # 当前游戏、游戏列表
├── useModStore         # mod 列表、启用状态、元数据
├── usePresetStore      # preset/pack 数据、当前激活 preset
├── useKeyStore         # 快捷键数据、覆写
├── useFilterStore      # 筛选状态（隔板展开、选中标签、搜索词）
├── useDesktopStore     # 桌面布局（每个物件的位置、大小、展开状态）
└── useUIStore          # 主题、动画强度、窗口状态
```

**数据流：**

```
Tauri Backend (Rust chunk-store / TS Core)
        ↕ invoke / event
Tauri API Bridge
        ↕
Zustand Stores ←──→ Tauri Event (多窗口同步)
        ↕
React Components (Desktop 上的各种卡片)
```

## 八、技术选型

| 需求 | 选择 | 理由 |
|------|------|------|
| UI 框架 | React 18+ | AI 工具链兼容 |
| 桌面框架 | Tauri 2 | 多窗口、Rust 后端 |
| 状态管理 | Zustand | 轻量、TS 友好 |
| 样式 | Tailwind CSS + CSS Modules | 原子化 + 作用域 |
| 3D 卡片 | CSS 3D Transforms | 倾斜、翻转、厚度 |

| 动画 | Framer Motion | 亮灭、翻转、展开、stagger |
| 无头组件 | Radix UI | 气泡、菜单可访问性 |
| 图标 | Lucide React | 统一风格 |
| 国际化 | i18next | 中英文 |
| 多窗口 | Tauri WebviewWindow API | 遥控卡、详情窗 |
| 桌面布局持久化 | Zustand persist | 记住用户的桌面摆放 |

## 九、视觉规范

### 卡片材质
- **哑光**：微妙内阴影 + 线性渐变表面纹理
- **厚度**：底部加深阴影模拟 2-3px 厚度
- **三档阴影**：静止(浅) → hover(中) → 拖拽(深)

### 光泽
Hover 时 radial-gradient 跟随鼠标扫过卡片表面。

### 色调
- 深色主题默认
- 强调色跟随游戏：ZZZ=橙黄, GI=青, HSR=紫
- 启用态卡片用强调色发光边框

### 动效预算

| 动画 | 时长 | 缓动 |
|------|------|------|
| 卡片翻转 | 400ms | ease-out |
| 亮灭切换 | 300ms | ease-in-out |
| 筛选沉浮 | 500ms | spring |
| 拆箱 | 600ms | ease-out |
| 盒子开合 | 400ms | ease-in-out |
| 隔板展开/收起 | 300ms | ease-out |

### 字体
- 标题：HarmonyOS Sans / 思源黑体，加粗
- 正文：系统 sans-serif
- 卡片背面编辑：可选手写风格字体

## 十、旧版继承清单

| 旧版组件 | 新版对应 | 变化 |
|---------|---------|------|
| `SectionSelector` | 不再需要 | 桌面无 section 切换 |
| `SectionSlider` | 不再需要 | 桌面无水平滑移 |
| `LeftIndex` | `DividerCard` | 从固定侧边栏 → 桌面上可拖拽的隔板卡 |
| `TagSearch` | `SearchCard` + `DividerCard` 标签隔板 | 从独立组件 → 桌面物件 |
| `FlippableElement` | `ModCard` 翻转 | 保留并增强 3D 厚度 |
| `AnimatedElement` | 拖拽物理效果 | 保留物理旋转模拟 |
| `ToggleableCardElement` | `ModCard` 亮/暗态 | 发光/暗淡替代简单高亮 |
| `DragableCardList` | `Desktop` 自由拖放 | 从列表 → 自由桌面 |
| `dialogModInfo2` 悬浮编辑 | `CardBack` 就地编辑 | 从对话框 → 卡片背面 |
| `BergerFrame` 三段式布局 | `Desktop` 全屏桌面 | 从框架布局 → 自由桌面 |
