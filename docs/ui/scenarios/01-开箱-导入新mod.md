# 场景一：开箱 —— 导入新 Mod

## 场景描述

用户从 GameBanana / Nexus 下载了一个新 mod 的压缩包，想把它加进管理器。

## 用户故事

> 我在 GameBanana 看到一个超好看的艾莲泳装 mod，下载好了压缩包。我把它拽进管理器，就像拆一个快递盒——拆开、看看里面是什么、给它贴个标签、放进我的收藏里。

## 交互流程

### 第一步：投递

用户把压缩包/文件夹拖进应用窗口。

应用窗口是全屏半透明的桌面。拖进来的瞬间，鼠标位置出现一个**未拆的小包裹**——带牛皮纸质感的小方块，上面用马克笔风格的手写体标着文件名。

```
       ╭───────────────╮
       │ ┈┈┈┈┈┈┈┈┈┈┈  │
       │  📦           │
       │  EllenSwim.zip│
       │ ┈┈┈┈┈┈┈┈┈┈┈  │
       ╰───────────────╯
```

包裹落在桌面上，有一个轻微的弹跳落地感。

### 第二步：拆箱

用户点击这个包裹（或者向上滑动），包裹**撕开**——纸片向两边翻开的动画。

里面露出的不是文件列表，而是一张**空白的新卡片**和一堆**散落的素材碎片**（代表文件）。

```
  纸片翻开
  ╱              ╲
 ╱   ┌────────┐   ╲
╱    │        │    ╲      ∘ 24个文件
     │  空白  │           ∘ 52.3 MB
     │  新卡  │           ∘ 
     │        │           ✓ 角色: 艾莲
     │  ┄┄┄┄  │           ✓ 快捷键: F1
     │  [写卡] │           ⚠ 冲突: 艾莲v2
     └────────┘
```

右侧显示自动分析的结果——角色识别、快捷键检测、冲突检测，像一张随包裹附带的"检验报告"小纸条。

**符合直觉且有趣**：拆快递/拆盲盒是人人都有的生活经验。撕开包装的动画给予满足感。"发现里面是什么"制造了一个微小的惊喜时刻——尤其是自动识别出角色的瞬间。

**高效**：用户拖入即触发，自动分析在拆箱动画播放的 300ms 内在后台完成。拆完直接看到结果，零等待感。

### 第三步：写卡

空白新卡是这次导入的核心交互。用户点击"写卡"，卡片翻转到背面，变成一个**可编辑的表单卡片**——但它不是一个传统表单，它看起来就像在一张实体卡片的背面手写信息。

```
  ┌──────────────────────┐
  │ ╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌ │
  │  名称: EllenSwim_v1  │  ← 可编辑，打字时有打字机动效
  │ ╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌╌ │
  │  角色: [艾莲 ▾]      │  ← 自动填充，可修改
  │                       │
  │  标签: 〔泳装〕〔+〕  │  ← 标签像贴纸一样贴上去
  │                       │
  │  备注:                │
  │  ┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈  │  ← 手写风格的备注区域
  │  ┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈┈  │
  │                       │
  │     [跳过]   [完成✓]  │
  └──────────────────────┘
```

添加标签时，标签像一张小贴纸从下方"啪"地贴到卡片上。删除标签时，贴纸被"撕下来"。

**符合直觉且有趣**：在卡片背面写字、贴标签，就像整理实体收藏卡时在背面做笔记。每一个编辑动作都有触感反馈（微动画）。

**高效**：可以直接点"跳过"——卡片会用自动识别的信息填好默认值。想精细管理的用户可以慢慢写卡，不想管的用户一秒完成。

### 第四步：收纳

写卡完成后，卡片翻回正面——封面图已经填好了（自动检测 preview 图片），角色名和 mod 名显示在底部。

这是一张完整的收藏卡了。

```
  ┌──────────────────────┐
  │ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │
  │ ▓▓▓▓▓ 艾莲泳装 ▓▓▓▓ │
  │ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │
  │ ▓▓▓▓▓▓▓ 预览图 ▓▓▓▓ │
  │ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │
  ├──────────────────────┤
  │  艾莲 · EllenSwim    │
  │  〔泳装〕        [●] │
  └──────────────────────┘
```

现在用户面前有这张新卡片，桌面上还散落着原来的其他卡片。

系统轻柔地提示：**"放到哪里？"**

- 直接丢在桌面上（加入 mod 列表，不归入任何 Pack）
- 拖到一个已有的卡叠上（加入 Pack）
- 拖到收纳盒里（加入 Preset）

卡片滑入目标位置的动画——如果是桌面，它飘到网格中自己的位置；如果是卡叠，它"嗒"地叠到最上面；如果是收纳盒，它滑入盒中。

**符合直觉且有趣**：拆完快递之后，自然的下一步就是"放到该放的地方"。整理收纳的行为本身就有整理癖的满足感。

**高效**：如果用户不想选，回车或点击空白处，卡片自动归入默认位置（桌面 + 自动按角色归类）。后台同时完成 chunk store 归档。

## 全流程时间

| 步骤 | 乐趣版（慢慢享受） | 效率版（一切默认） |
|------|--------------------|--------------------|
| 拖入 | 0.5s | 0.5s |
| 拆箱动画 | 0.8s | 可关闭，直接出卡 |
| 写卡 | 10-30s（自己编辑） | 0s（跳过） |
| 收纳 | 2s（选位置） | 0s（自动归位） |
| **总计** | **~35s** | **~1.5s** |
