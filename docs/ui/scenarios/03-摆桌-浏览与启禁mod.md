# 场景三：摆桌 —— 浏览与启用/禁用 Mod

## 场景描述

用户打开应用，想看看自己有哪些 mod，启用几个、禁用几个，准备开始玩游戏。

## 用户故事

> 我打开管理器，面前是我的"卡片桌"——所有 mod 像收藏卡一样铺开。我想给艾莲换个泳装 mod，把旧的那个禁了，新的启用。然后看看有没有冲突，没问题就开玩。

## 核心意象：一张大桌子

整个应用是一张半透明的大桌面。卡片"铺"在桌上。

不是传统的"文件管理器列表"，而是**你在桌上铺开所有收藏卡**的那种视觉。

### 启用/禁用的卡片视觉差异

这是最核心的视觉设计——用户一眼就能分辨哪些 mod 是开的、哪些是关的。

**启用状态（亮牌）**：
- 卡片正常亮度
- 底部有一圈柔和的发光边框（主题强调色）
- 卡片微微高于桌面（阴影更深 = 更"浮起"）
- 右下角有一个微小的光点 ●

**禁用状态（暗牌）**：
- 卡片整体降低亮度（加一层半透明暗色覆盖层）
- 没有发光边框
- 卡片"贴在"桌面上（阴影很浅 = 很"贴"桌面）
- 右下角光点熄灭 ○

**冲突状态（警告）**：
- 卡片边缘有橙红色脉冲光（呼吸灯效果）
- 右上角出现 ⚠️ 小徽标

### 启用/禁用操作

不需要翻到背面。直接在正面完成。

**方式A：点亮/熄灭**

单击一张暗牌 → 卡片**亮起**动画（从暗到亮，光从中心向外扩散，发光边框渐入）。
单击一张亮牌 → 卡片**熄灭**动画（光收缩到中心消失，发光边框渐出，整体暗下来）。

就像按下一个物理开关，但用灯光效果表达。

**方式B：滑动切换**

在卡片上做一个短促的向下划动 → 熄灭（想象把灯开关往下按）。
向上划动 → 亮起。

**方式C：拖到"启用区/禁用区"**

桌面底部有两个隐形区域——拖卡片到左下角（暗区域图标 ○）= 禁用，拖到右下角（亮区域图标 ●）= 启用。

### 筛选与桌面变化

当用户在左侧筛选器中点击"艾莲"：

不是切换页面，而是**桌面上发生了变化**：
1. 不匹配的卡片缩小 + 淡出 + 下沉到桌面以下（像沉入桌面的动画）
2. 匹配的卡片滑动归拢到一起，重新排列
3. 整个过程像在桌面上拨开不相关的卡片，只留下想看的

取消筛选时，之前沉下去的卡片重新浮上来，回到原位。

**符合直觉且有趣**：像在桌上翻找特定卡片——把不要的推到一边，想看的拉过来。

### 冲突处理

当用户启用一张卡片，导致与另一张已启用的卡片冲突时：

1. 新启用的卡片亮起
2. 冲突的那张卡片开始**抖动** + 边缘泛红脉冲
3. 两张卡片之间画出一条隐约的红色连线，标注"冲突"
4. 弹出一个小气泡（不是模态对话框！）：

```
     ╭────────────────────────────╮
     │ ⚠️ 与 EllenThicc 冲突       │
     │                             │
     │ 两者都修改了 EllenBody      │
     │                             │
     │  [禁用它]  [共存*]  [无视]  │
     ╰────────────────────────────╯
     
     * 共存 = 保持两个都启用，后果自负
```

这个气泡是一个浮在桌面上的小纸条——不是弹窗，不打断体验。用户如果不理它，几秒后它会缩小为一个 ⚠️ 角标留在卡片上。

### 快速启动游戏

桌面右下角有一个"启动"按钮——它的外形像是一张特殊的发光卡片：

```
  ╔══════════════╗
  ║  ▶ 启动游戏  ║  ← 特殊卡片，主题色发光
  ║              ║
  ║  12 mod 就绪 ║
  ╚══════════════╝
```

点击后，所有已启用的卡片做一个"集合"动画（所有亮牌快速缩小飞向这个按钮），然后游戏启动。

## 符合直觉且有趣

- 桌面隐喻：人人都有在桌上铺开卡片/照片找东西的经验
- 亮/暗 表达启用/禁用——比 checkbox 更直观，一眼看整体
- 冲突不打断操作——一个小纸条提醒，你可以选择不理会
- 筛选 = 拨开其他卡片——不是"进入筛选模式"
- 启动前的"集合"动画给一个小仪式感

## 高效

- 单击即切换启用/禁用，零思考成本
- 启用/禁用有强烈视觉反馈，不需要看文字确认
- 筛选不需要页面跳转，桌面连续变化
- 冲突提示非阻断，可以忽略
- 多选（框选/Ctrl+点击）→ 批量点亮/熄灭
